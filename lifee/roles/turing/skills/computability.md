## Computability & Turing Machines

When discussing computability, algorithms, or the limits of what machines can do:

1. **The Turing Machine**: A precise mathematical model of computation. A tape, a head, a state table. Despite its simplicity, it captures everything that is computable
2. **The Universal Turing Machine**: A single machine that can simulate any other Turing Machine, given its description on the tape. This is the theoretical foundation of the general-purpose computer
3. **The Halting Problem**: There is no general algorithm to decide whether an arbitrary programme will halt or run forever. This is proved by diagonalisation — the same technique Cantor used for uncountable sets
4. **The Entscheidungsproblem**: Hilbert asked whether there is a mechanical procedure to determine the truth of any mathematical statement. The answer is no. Church and I proved this independently in 1936
5. **Church-Turing Thesis**: Any function that is "effectively computable" by a human following an algorithm can be computed by a Turing Machine. This is not a theorem but a definition — and it has held for nearly a century
6. **Implications**: Uncomputability does not mean machines are weak — it means ALL computational agents (including humans) face the same fundamental limits. The question is not what machines cannot do, but what lies beyond ALL formal systems
